## How this documentation is organized

This site follows **[Diátaxis](https://diataxis.fr)**, the widely-adopted documentation
framework that splits docs into four types by what the reader needs *right now* — learning vs.
doing, practical vs. theoretical. Each page is exactly one type. Architecture is described with
the **[C4 model](https://c4model.com)**, and decisions are recorded as **ADRs**.

## Status legend

Throughout these docs, features carry one of these tags:

- ✅ **Implemented** — exists in the codebase today.
- 🟡 **Designed** — specified here, not yet built.
- 🔭 **Considered** — evaluated, intentionally deferred (with rationale).
